原码、反码、补码

搞懂什么是原码？什么是反码？什么是补码？

数字6按位取反为什么会是-7呢？那么就得知道计算机是怎么识别数值6的，众所周知计算机只能识别二进制的数据，什么八进制、十进制、十六进制，都得转换成二进制计算机才能认识，因此在计算机识别数值6的时候会先把数值6转换成二进制。
另外，计算机又分什么8位、16位、32位，这些是什么呢？这些说的是计算机的处理器(也就是CPU)处理信息的方式，下面以8位计算机来讲解数字6按位取反为什么会是-7，在那之前先了解一下什么是原码、反码和补码。

原码：将最高位作为符号位(0表是正，1表示负)，其他数字位代表数值本身的绝对值。这也是计算机表示数字的方式
有了数值的表示方式就可以对数值进行运算，但是很快就发现带符号位的原码进行乘除运算时结果正确，而在加减运算的时候会出现问题如下：

为了解决原码的缺陷于是就有了反码。
反码：如果是正数，则表示方法和原码一样，如果是负数，则保留符号位1，然后将这个数字的原码按照每位取反。
         
在反码中还是存在问题，两个正数的加法运算是没有问题的，问题出在带符号位的负数身上，对于除符号位外的其余各位逐位取反就产生了反码，反码的取值空间和原码相同且一一对应，下面是反码的减法运算：

第一行我们人计算是没问题的，但是第二行计算机计算的时候问题就来了，计算机中有-0这个数值的存在，但是正常情况下，人们计算是没有-0这个存在的，0是没有正负之分的。所以原码和反码在计算过程中，都存在问题，所以补码就出现了。
补码：整数补码保持不变，也就是不进行补码，正数的原码、反码、补码都是一样的。负数补码就是对反码加一，在补码中用(-128)代替了(-0)，所以补码表示范围为(-1280127)，共256个，而上面说的原码和补码的范围是(-127-127)不包含128。
补码概念：补码是计算机数据的一般方式，其规则为：如果是正数，则表示方法和原码一样，如果是负数，则将数字的反码加上1(相当于将原码数值按位取反后再对其的加1)

第一行：结果为0没问题
第二行：正整数1变成原码后是00000001，负整数1，变成反码后是111111110然后进行补码就变成了11111111,然后转回原码就是00000000，也就是0
第三行：结果为-1没问题
第四行：正整数1变成原码后是00000001，负数要把原码转反码，所以负整数2的原码是00000010，反码就是11111101，然后再进行补码，-2补码是11111110，1的原码加-2的(反码 补码)，就等于-1然后-1的原码是00000001，[反码 补码]就是111111111也就是-1,，结果没问题
回到正题6按位取反后为什么是-7
第一步：6的原码是00000110
第二步：对6进行按位取反的操作，那么6 反码 补码后 是11111001
第三步：6在按位取反也就是进行反码的时候还进行了补码操作也就是加一，所以还需要减一，这时候6的反码就是11111000，别急还没完，因为反码是给计算机识别的，不是给我们识别的，所以我们还需要再进行一次按位取反操作，把6再改回我们可以识别的原码
第四步：对6的反码进行按位取反，这里误以为6的反码11111001进行按位取反后是00000110的那就错了，这样想那按位取反意义何在搞了大半天6还是那个6，取反！取反！取反！当然是代表负数的符号位1保持不变那样就是取反了，取反后的结果是10000111，算一下就是-7
扩展：8位计算机、16位计算机的区别在于，8位计算机所能计算的最大值就是2的八次方，16位计算机所能计算的最大值就是2的十六次方，一次类推，32、64等。
